# 📁 دليل تثبيت ورفع تطبيق "حصّالة الذهب والادخار" على الاستضافة (Hostinger / cPanel)

تم تزويد التطبيق بنظام متكامل لإدارة البيانات سحابياً والتحقق من حسابات المستخدمين ومزامنة المعاملات والمدخرات والميزانيات تلقائياً مع قاعدة بيانات **MySQL / MariaDB**، مع توفر وضع مريح لحفظ البيانات محلياً (Local Storage) في المتصفح في حال عدم تشغيل قاعدة البيانات السحابية.

هذا الدليل يشرح بالتفصيل كيفية رفع التطبيق على استضافة **Hostinger** أو أي لوحة تحكم **cPanel** مع إنشاء وتهيئة قاعدة البيانات بالكامل.

---

## 🚀 ملخص خطوات التثبيت السريع
1. **بناء المشروع**: إنتاج الملفات النهائية المجهزة للاستضافة.
2. **إنشاء قاعدة البيانات**: إعداد قاعدة بيانات MySQL جديدة في الاستضافة.
3. **رفع الملفات**: رفع الأكواد إلى مجلد الاستضافة الرئيسي.
4. **تكوين البيئة (`.env`)**: إعداد مفتاح الذكاء الاصطناعي وبيانات قاعدة البيانات.
5. **تثبيت جداول قاعدة البيانات**: تشغيل السيناريو التلقائي `install.js` أو استيراد `database.sql`.
6. **تشغيل الخادم**: إطلاق التطبيق على الويب.

---

## 📋 الخطوات بالتفصيل الممل

### الخطوة الأولى: بناء المشروع (Build)
قبل الرفع على الاستضافة، تحتاج إلى بناء الكود المصدري ودمج خادم Express مع واجهة React.
في سطر الأوامر (Terminal) بمجلد المشروع، قم بتشغيل الأمر التالي:
```bash
npm run build
```
* هذا الأمر سينشئ مجلداً جديداً باسم `dist` يحتوي على ملفات الواجهة الأمامية المصغرة، وسيقوم بضغط وترجمة خادم Node.js في ملف واحد داخل `dist/server.cjs` ليعمل بكفاءة فائقة على الاستضافة وبدون مشاكل في مسارات الاستيراد.

---

### الخطوة الثانية: إنشاء قاعدة بيانات MySQL جديدة على الاستضافة
1. ادخل إلى حسابك في **Hostinger hPanel** (أو لوحة cPanel).
2. ابحث عن قسم **قواعد البيانات (Databases)** وافتح **قواعد بيانات MySQL**.
3. قم بإنشاء قاعدة بيانات جديدة:
   - **اسم قاعدة البيانات (Database Name)**: مثلاً `u123456789_hassala`
   - **اسم مستخدم قاعدة البيانات (Database User)**: مثلاً `u123456789_user`
   - **كلمة المرور (Database Password)**: اختر كلمة مرور قوية جداً.
4. قم بحفظ هذه التفاصيل جانباً، سنحتاجها لاحقاً في ملف التكوين.

---

### الخطوة الثالثة: رفع الملفات إلى الاستضافة
1. في لوحة تحكم الاستضافة، افتح **مدير الملفات (File Manager)** الخاص بنطاقك.
2. ادخل إلى المجلد الأساسي للموقع (عادةً ما يكون `public_html` أو مجلد النطاق الفرعي المخصص).
3. قم بضغط ملفات ومجلدات المشروع في جهازك (باستثناء مجلد `node_modules` ومجلد `dist` الخارجي - **فقط ارفع ملفات جذر المشروع مثل package.json، install.js، database.sql، ومجلد dist الناتج عن عملية البناء**).
4. ارفع ملف الـ ZIP إلى الاستضافة وقم بفك الضغط عنه مباشرة داخل المجلد الرئيسي.
5. تأكد من أن ملف `package.json` ومجلد `dist` هما في الجذر الرئيسي للاستضافة مباشرة.

---

### الخطوة الرابعة: إعداد ملف المتغيرات البيئية (`.env`)
في المجلد الرئيسي لموقعك على الاستضافة، قم بإنشاء ملف جديد باسم `.env` (تأكد من وجود النقطة في البداية) واكتب بداخله القيم التالية مستبدلاً إياها ببياناتك الفعلية:

```env
# منفذ تشغيل الخادم (هوستنجر تتعامل مع المنافذ تلقائياً، يرجى تركه 3000 أو المنفذ المحدد من هوستنجر)
PORT=3000

# مفتاح الوصول للذكاء الاصطناعي Gemini API (مهم لعمل المستشار المالي والدردشة الذكية)
GEMINI_API_KEY=ضع_هنا_مفتاح_جيميني_الخاص_بك_AIzaSy

# تفاصيل الاتصال بقاعدة بيانات MySQL التي قمت بإنشائها في الخطوة الثانية
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=u123456789_user
DB_PASSWORD=ضع_هنا_كلمة_مرور_قاعدة_البيانات_القوية
DB_NAME=u123456789_hassala
```

---

### الخطوة الخامسة: تهيئة وتثبيت جداول قاعدة البيانات بالكامل
لقد وفرنا لك **طريقتين** لإنشاء جداول قاعدة البيانات وتعبئتها بالبيانات التجريبية الافتراضية بسهولة تامة:

#### **الطريقة الأولى (التثبيت التلقائي الذكي - موصى بها):**
إذا كان لديك صلاحية الوصول إلى الطرفية (Terminal) في استضافتك أو كنت تستخدم مدير تطبيقات Node.js في هوستنجر، قم بتشغيل الأمر التالي لمرة واحدة فقط لتثبيت الجداول تلقائياً:
```bash
node install.js
```
* سيقوم السكربت بالاتصال بقاعدة بياناتك، إنشاء الجداول الثلاثة (`users`, `transactions`, `category_budgets`)، وإدراج مستخدم افتراضي بالبريد الإلكتروني `admin@hassala.com` مع كلمة مرور `123456` وخمسة معاملات تجريبية وميزانيات كاملة للبدء فوراً.

#### **الطريقة الثانية (الاستيراد اليدوي عبر phpMyAdmin):**
1. من لوحة استضافة هوستنجر، افتح أداة **phpMyAdmin** وادخل إلى قاعدة البيانات الخاصة بك.
2. انقر على تبويب **استيراد (Import)** في الأعلى.
3. اختر الملف المرفق المسمى **`database.sql`** من جهازك.
4. اضغط على زر **انطلاق (Go)** أو **استيراد** في الأسفل.
* سيتم بناء كافة الجداول وتهيئة الحسابات والمعاملات التجريبية فوراً!

---

### الخطوة السادسة: تشغيل التطبيق على الاستضافة
1. **استخدام Node.js App Manager في هوستنجر**:
   - ابحث عن **Node.js** في لوحة التحكم hPanel.
   - قم بإعداد تطبيق Node.js جديد وحدد:
     - **مجلد التطبيق**: المجلد الذي رفعت فيه الملفات (مثلاً `/public_html`).
     - **ملف الدخول الأساسي (Application Entry File)**: حدده ليكون `dist/server.cjs`.
     - **نسخة Node.js**: اختر النسخة `18` أو `20` أو أعلى.
   - انقر على **تثبيت (Install)** ثم **تشغيل (Start)**.
2. **استخدام سطر الأوامر في VPS أو السيرفرات الخاصة**:
   شغل الأمر التالي بداخل المجلد لتشغيل السيرفر بشكل دائم في الخلفية:
   ```bash
   # تثبيت حزم التشغيل
   npm install --production
   
   # تشغيل السيرفر بشكل دائم
   npm run start
   ```

## 🛠️ الدعم الفني وحل المشكلات الشائعة
- **المستشار الذكي يعطي خطأ أثناء الدردشة؟**
  تأكد من إدخال مفتاح `GEMINI_API_KEY` بشكل صحيح وبدون فواصل في ملف `.env` في الاستضافة، وتأكد من أن السيرفر متصل بالإنترنت.
- **التطبيق يعطي "وضع حفظ محلي 💾" بدلاً من "سحابي ☁️" في الشريط الجانبي؟**
  هذا يعني أن خادم Express لم يتمكن من الاتصال بقاعدة بيانات MySQL. يرجى مراجعة بيانات الاتصال المكتوبة في ملف `.env` والتأكد من تطابقها تماماً مع قاعدة البيانات التي أنشأتها في هوستنجر.
- **كيف يتم أخذ نسخ احتياطية؟**
  يمكنك دائماً أخذ نسخة احتياطية من قاعدة البيانات عبر خيار Export في phpMyAdmin، كما يوفر التطبيق ميزة فريدة من نوعها لتحميل ملف النسخة الاحتياطية بصيغة JSON بضغطة زر واحدة من تبويب "الرفع على الاستضافة" لاستعادتها في أي وقت!
